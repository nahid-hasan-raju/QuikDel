# Data Format Reference

This document describes every intermediate data file produced and consumed by the QuikDel pipeline.

---

## Directory: `Census Data/`

| File | Format | Description |
|---|---|---|
| `census_tract_data.geojson` | GeoJSON | Census tract polygons with producer/consumer counts and bordering tract info. Generated by Notebook 01. |

---

## Directory: `Hotspot Data/Data-<min_children>-<ratio>/`

The subfolder name encodes the clustering parameters:  
- `<min_children>` = minimum cluster size = `ratio - 3`  
- `<ratio>` = superspot-to-hotspot ratio (e.g., 5, 10, 15)

| File | Format | Description |
|---|---|---|
| `graphhopper_dataframe.csv` | CSV | One row per hotspot (census tract). Columns include `GEOID`, superspot assignment, list of child tracts (`children`), and average producer/consumer counts. |
| `distance_adjacency_matrix.npy` | NumPy `.npy` | N×N matrix of road distances (meters) between all connected hotspot pairs. Unreachable pairs set to `1e9`. |
| `time_adjacency_matrix.npy` | NumPy `.npy` | N×N matrix of travel times (seconds) between all connected hotspot pairs. |
| `map_geoid_index.json` | JSON | Bijection mapping Census GEOID strings → integer hotspot indices. **Must be consistent across all pipeline stages.** |

---

## Directory: `Q Tables/Data-<min_children>-<ratio>/`

| File | Format | Description |
|---|---|---|
| `Q_boltzman.npz` | NumPy `.npz` | Compressed archive of all intra-cluster Q-tables. Keys are integer superspot indices. Each value is a 3D array `[destination, state, action]`. |
| `index_to_Q.json` | JSON | Maps global hotspot index → intra-cluster Q-table local index. |
| `Q_to_index.json` | JSON | Inverse of `index_to_Q`. Nested: `{superspot_idx: {local_idx: global_idx}}`. |
| `index_to_Q_super.json` | JSON | Maps global superspot index → inter-cluster Q-table local index. |
| `Q_super_to_index.json` | JSON | Inverse of `index_to_Q_super`. |
| `tract_group_lists.json` | JSON | Maps superspot index → list of child hotspot global indices. Key `-1` holds all superspot indices. |
| `tract_super_dict.json` | JSON | Maps each hotspot global index → its superspot global index. |

---

## Directory: `Results/Data-<min_children>-<ratio>/`

Result files follow the naming convention:  
`<type>_<load>-<run_id>.json`

Where:
- `<type>` is one of `results` (QuikDel), `baseline1`, or `baseline2`
- `<load>` is the delivery load parameter (e.g., `0300` = 300 deliveries/hour)

| Key in JSON | Description |
|---|---|
| `success_rate` | Fraction of deliveries completed within their time limit |
| `distances_tagged` | List of `[distance_m, was_shared, was_successful]` per delivery |
| `times_tagged` | List of `[time_s, was_shared, was_successful]` per delivery |
| `distances` | All delivery distances (including failed) |
| `times` | All delivery times (including failed) |
| `hops` | Average number of hotspot hops per successful delivery |
| `shares` | Total number of path-sharing events |
| `CDV` | Number of core delivery vehicles used |
| `PDV` | Number of peripheral delivery vehicles used |

---

## Hotspot-to-Superspot Ratio Quick Reference

| Ratio | `min_children` | Approximate cluster size |
|---|---|---|
| 1:5 | 2 | ~5 tracts per cluster |
| 1:10 | 7 | ~10 tracts per cluster |
| 1:15 | 12 | ~15 tracts per cluster |
